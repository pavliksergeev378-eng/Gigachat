Тест Pyodide + math-agent логики

Изолированный тест без LLM и n8n. Проверяет: stdlib Python, математический комплект (sympy/numpy/scipy/mpmath), полный шаблон «Генерация sympify», JS-сторону (preprocessing запятой, extractFinalLine с regex для опечатки «ОТВИД»). Первые 14 тестов — stdlib, дальше пакеты, в конце — JS-only.

Что это проверяет:
• stdlib (math, statistics, decimal, fractions) — базовый Python в браузере
• Бесконечный цикл + восстановление — worker корректно убивается по таймауту и пересоздаётся
• sympy / numpy / scipy / mpmath — .whl файлы в Agents/lib/pyodide/ загружаются, новые пакеты работают (первая загрузка ~10-30 сек для каждого)
• Полный шаблон «Генерация sympify» — ровно тот Python, что генерит сервер для pure_expr ветки. Если этот тест зелёный, sympify-путь в math-agent тоже будет работать
• JS-сторона — preprocessing русской запятой (с lookbehind), extractFinalLine для «ОТВЕТ», «ОТВИД» (опечатка GigaChat) и диагностических строк после маркера

Если все тесты зелёные — Pyodide-цепочка + математический бандл + JS-логика готовы. В офисе после подключения LLM/Postgres math-agent должен заработать с первой попытки.

Если красные:
• stdlib красный → Pyodide не грузится. Проверь Agents/lib/pyodide/: pyodide.asm.wasm, python_stdlib.zip
• sympy/numpy/scipy красные → отсутствуют .whl файлы или SHA256 не совпадает с pyodide-lock.json. Проверь, что у тебя 5 файлов: sympy-1.12-...whl, mpmath-...whl, numpy-...whl, scipy-...whl, openblas-...zip
• JS-тесты красные → браузер не поддерживает lookbehind regex (нужен Chrome 62+, Firefox 78+)

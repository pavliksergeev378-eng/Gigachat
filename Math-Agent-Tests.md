# Тестирование Math-агента в офисе (v2)

Версия после двух раундов аудита (коммиты `25c8f87`, `d9d8927`). Перед прогоном:

1. **Перезалить `Workflow/math-agent.json` в n8n** — n8n кеширует workflow, простая замена файла без re-import не сработает. Открыть UI → Import workflow → выбрать файл → Replace → Save → Deactivate → Activate.
2. Активны credentials Postgres + GigaChat (без auth).
3. Pyodide бандл лежит в `Agents/lib/pyodide/`.
4. Фронт обновлён из git pull, страница перезагружена (Ctrl+F5 для сброса кеша).

Колонки `Ветка` — куда классификатор направит запрос:
- **PURE** — pure_expr (sympify без LLM, детерминированно)
- **MATH** — LLM генерирует Python (с sympy/numpy/scipy)
- **REEX** — reexplain (переиспользование code/raw_result из БД)
- **GEN** — general chit-chat (без вычислений)
- **VAL** — отклонено валидацией / security
- **EXEC** — стадия исполнения кода фронтендом (внутренняя)

Каждый тест — уникальное выражение/задача; повторов нет.

## Если что-то упало

1. **DevTools → Network → `/webhook/math` или `/math-explain` → Response**:
   - `{response: "GigaChat-LLM недоступен"}` → LLM не запущен / неправильные creds
   - `{response: "Pyodide worker не инициализировался"}` → .whl файлов нет в `Agents/lib/pyodide/`
   - `{response: "Запрос отклонён: ..."}` → валидация Phase 2 отказала
   - `{response: "Не удалось решить — Python завершился..."}` → Python exception (часто из-за LLM-кода)
   - `{response: "Ответ: ошибка парсинга"}` → sympify не понял (pure_expr ветка)
   - `{response: "Модуль не разрешён: ..."}` → security validator (КР-5 семья)
   - `{response: "Запрещены dunder-методы..."}` → security validator (КР-A/КР-6)
   - `{response: "Запрещены строки из подчёркиваний..."}` → новая КР-6 защита

2. **DevTools → Console**: ошибки от Pyodide worker, regex syntax-errors из n8n (если новые `\\\\` неправильно проинтерпретированы).

3. **n8n → Executions** → найти упавший узел. Если упал «Классификатор» — лог должен содержать `lower.match is not a function` или подобное. Если упал «Проверка кода» — проблема с regex/decoded.

4. **Откат (если массово ложится)**:
   ```

   git push origin main
   ```
   Импортнуть старую версию workflow в n8n.

5. **Скрин Network + Console + n8n Execution** → присылайте.

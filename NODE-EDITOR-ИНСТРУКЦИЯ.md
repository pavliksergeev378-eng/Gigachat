# Инструкция: Node Editor — визуальный конструктор агентов

## Что это

**Node Editor** — это визуальный конструктор, в котором можно **наглядно собрать пайплайн** из агентов и инструментов: перетащить нужные компоненты на холст, соединить их, настроить параметры и экспортировать конфигурацию для n8n.

**Где находится:** `Agents/node-editor.html`

---

## Как открыть

**С дашборда:** перейдите на `GigaChat-Platform.html` → раздел **«Конструктор»** → карточка **«Node Editor»**.

**Напрямую:** откройте в браузере:
```
http://localhost:8765/Agents/node-editor.html
```

> **Важно:** Node Editor, как и все агенты, требует авторизации. Если вы не залогинены — будет редирект на `/login.html`.

---

## Как пользоваться: пошагово

### Шаг 1. Перетащите агента или инструмент на холст

Слева находится **палитра** с тремя разделами:

| Раздел | Что внутри |
|--------|-----------|
| 🤖 **Агенты** | Chat, RAG, Math, Prompt, Plane |
| 🛠 **Инструменты** | Поиск ФИО, OCR+Таблицы, Document Loader, Text Extractor, Table Merger, File Merger, Организация обращения, Compliance |
| ⚙️ **Служебные** | Router (условная маршрутизация) |

Просто **захватите элемент мышкой** и перетащите на серый холст.

### Шаг 2. Перемещайте ноды по холсту

Кликните на ноду и **перетащите** — она свободно двигается по холсту.

### Шаг 3. Соедините ноды

У каждой ноды есть **входные порты** (слева, синие точки) и **выходные порты** (справа, розовые точки).

1. Нажмите и **тяните от выходного порта** (розовая точка)
2. Отпустите на **входном порту** другой ноды (синяя точка)
3. Появится SVG-линия — соединение создано

**Удалить соединение:** кликните на линию.

### Шаг 4. Настройте ноду

**Двойной клик** по ноде → откроется панель настроек справа:

- **Название** — переименовать ноду
- **Описание** — краткое пояснение
- **Тег** — метка для группировки (например: `main`, `test`, `docs`)
- **Параметры (JSON)** — произвольные настройки, которые попадут в экспорт
- **🔗 Открыть агента** — кнопка открывает соответствующую HTML-страницу агента в новой вкладке

### Шаг 5. Сохраните схему

**💾 Save** (или Ctrl+S) — скачает файл `node-editor-schema.json`.

**📂 Load** — загрузите ранее сохранённую схему обратно.

### Шаг 6. Экспортируйте в n8n

**🚀 Export n8n** — генерирует JSON с webhook-адресами для n8n. Можете импортировать этот файл в n8n через **Import Workflow**.

---

## Горячие клавиши

| Клавиша | Действие |
|---------|----------|
| **Ctrl+S** | Сохранить схему |
| **Ctrl+=** | Приблизить |
| **Ctrl+-** | Отдалить |
| **Ctrl+0** | Сбросить масштаб |
| **Delete / Backspace** | Удалить выбранную ноду |
| **Esc** | Закрыть панель настроек / снять выделение |

**Zoom колесом мыши** при зажатом Ctrl.

---

## Для чего это нужно

1. **Визуализация пайплайна** — вместо того чтобы держать в голове цепочку «Chat → Router → OCR → Table Merger», вы видите её на экране
2. **Быстрая настройка** — не нужно заходить в n8n, чтобы поменять порядок или добавить новый шаг
3. **Документирование архитектуры** — сохранили JSON → отправили коллеге → он загрузил и видит ту же схему
4. **Экспорт в n8n** — конфигурация готова к импорту, не нужно пересобирать workflow вручную

---

## Известные ограничения

- Node Editor **не запускает** workflow'ы — он только генерирует конфигурацию. Для запуска нужен n8n или открытие агентов по отдельности.
- **Router** добавлен как служебная нода для ветвления, но в n8n-экспорте он обозначен как отдельный элемент.
- При загрузке схемы старые ID нод **пересоздаются** — соединения восстанавливаются, если порядок нод в файле соответствует оригиналу.
- Для нод без отдельной HTML-страницы (OCR+Таблицы, Router) кнопка **«Открыть агента»** недоступна.

---

## Примеры использования

**Простой:** Chat → Text Extractor (сначала извлекаем текст из документа, потом отправляем в чат)

**Средний:** Chat → Router → два пути:
- Route A → Math (вычисления)
- Route B → RAG (поиск по документам)

**Сложный:** Документ → OCR → Compliance Check → Org. Appeal (распознали → проверили → подали обращение)

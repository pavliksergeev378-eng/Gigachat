# Plane-агент — гайд тестирования чата (v2)

Гайд для ручного тестирования чата Plane-агента «как пользователь». Тесты идут
**от простого к сложному** и **зависят друг от друга по порядку** (что создаём в
начале — меняем и удаляем в конце), поэтому проходи сверху вниз, не вразнобой.

---

## 0. Подготовка (один раз перед стартом)

1. В Plane должны существовать два проекта: **«Общие задачи»** и **«Специальные задачи»**.
   Если их нет — заведи вручную через интерфейс Plane (агент проекты не создаёт, это нормально).
2. Открой Plane-агента и создай **новую сессию** (чистая история — многоходовые тесты на это рассчитывают).
3. Тестируй **из общего чата**, НЕ открывая доску конкретного проекта (чтобы не было «sticky-проекта»).
   Отдельный блок 13 — специально про sticky, там доску откроем.
4. Логинься под собой (имя из аккаунта подставляется в ответы — это ок).

---

## Как читать и фиксировать каждый тест

Формат теста: **Ввод** (что печатаешь) → **Ждём** (что должно произойти, глазами) → местами **❌ Баг, если** (явный признак провала).

🔑 **Главное правило тестирования действий.** Для всего, что *создаёт / меняет /
удаляет* задачу, **мало прочитать ответ агента** — открой доску проекта в Plane и
проверь, что действие **реально произошло** (задача появилась/изменилась/исчезла,
с правильным именем и в правильном проекте). Самый частый баг — «агент написал
„Создаю…“, а в Plane ничего нет» — ловится только так.

Фиксируй по каждому: `☐ OK` или `☐ FAIL`. Для FAIL запиши: **№ теста — что ждал → что получил** (по возможности скриншот). Список отдашь мне.

**Критичные провалы** (всегда баг): английские слова в ответе; видно `{"action":...}`
или ```json; сказал «создал/изменил/удалил», а в Plane нет изменения; выдуманный
проект/задача, которых нет; «Меня зовут <твоё имя>».

## 🔴 Блок R. Быстрый регресс — прогони ПЕРВЫМ

Это сценарии, которые недавно чинились. Если хоть один ❌ — сразу пиши, не проходя дальше.

**R1. Создание проекта — агент НЕ должен врать.**
Ввод: `Создай новый проект Маркетинг`
Ждём: текстом объясняет, что **проекты заводятся вручную через интерфейс Plane**, по API нельзя.
❌ Баг, если: пишет «Создаю проект Маркетинг» / «Создал проект» (проект при этом НЕ появляется — это враньё).

**R2. Создание задачи без проекта → ответ названием проекта (многоходовой).**
- Шаг 1 — ввод: `Создай задачу Позвонить Иванову`
  Ждём: спрашивает «В каком проекте создать задачу «Позвонить Иванову»? Доступные: …». Пока НЕ создаёт.
- Шаг 2 — ввод: `Общие задачи`
  Ждём: создаёт задачу **«Позвонить Иванову»** в проекте **«Общие задачи»** (карточка в чате).
  ❌ Баг, если: показывает список задач проекта / снова переспрашивает / «создаёт» задачу с ДРУГИМ именем / ничего не создаёт.
  Проверь в Plane: на доске «Общие задачи» появилась «Позвонить Иванову».

**R3. Несколько задач за один запрос (bulk).**
Ввод: `Создай 3 задачи: А, Б, В в Общих задачах`
Ждём: создаются **три** задачи — «А», «Б», «В» — все в «Общие задачи».
❌ Баг, если: создалась только «А» / создалась одна задача с именем «А, Б, В».
Проверь в Plane: на доске три отдельные задачи.

**R4. Приоритет по смыслу + без проекта (многоходовой).**
- Шаг 1 — ввод: `Срочно: упал сайт, нужен фикс`
  Ждём: спрашивает, в каком проекте создать (задача «Упал сайт, нужен фикс» или похоже).
- Шаг 2 — ввод: `Специальные задачи`
  Ждём: создаёт задачу в «Специальные задачи» с приоритетом **срочный (urgent)**.
  Проверь в Plane: задача есть, приоритет — Срочный.

**R5. Явный приоритет + без проекта (многоходовой).**
- Шаг 1 — ввод: `Создай задачу с высоким приоритетом "Документация"`
  Ждём: спрашивает, в каком проекте.
- Шаг 2 — ввод: `Общие задачи`
  Ждём: создаёт «Документация» в «Общие задачи». Приоритет в идеале — **высокий**.
  (Допустимо: если приоритет вышел обычный — отметь, это известное ограничение, не критично.)

**3.** `Покажи задачи проекта Общие задачи` → задачи этого проекта (карточками), НЕ список проектов.

## Блок 2. Идентичность и болтовня

**4.** `Кто ты?` → «Я Plane-агент» + кратко о функциях.
**5.** `Как тебя зовут?` → «Я Plane-агент» (НЕ твоё имя).
**6.** `Меня зовут Георгий` → «Запомнил, Георгий. Чем помочь?». ❌ Баг: «Меня зовут Георгий».
**7.** `Как меня зовут?` (сразу после теста 6) → «Георгий» (помнит из истории).
**8.** `Спасибо` → короткое «На здоровье. Что ещё?». ❌ Баг: повторяет прошлое действие.
**9.** `Хорошо` / `Ага` / `Окей` → «Готов помочь дальше, что делаем?». ❌ Баг: повтор действия.
**10.** `Какая сегодня погода?` → вежливо: «Я работаю с Plane, не с погодой».
**11.** `Реши уравнение 2x+3=7` → «Я Plane-агент, для математики — math-агент» (мягкий отказ).
**12.** `Не работает! Опять глюк` → уточняющий вопрос «что именно не так?», БЕЗ оправданий.

## Блок 4. Просмотр и поиск задач

**15.** Задачи проекта — варианты (каждая → задачи «Специальные задачи»):

**16.** `Покажи все задачи` / `Какие задачи?` → задачи по **всем** проектам (cross-project), без требования выбрать проект.
**17.** `Покажи задачи проекта Маркетинг2` (такого нет) → «Такого проекта нет, доступные: Общие задачи, Специальные задачи». ❌ Баг: выдумывает задачи/проект.
**18.** `Найди задачи про "хлеб"` → поиск по всем проектам, показывает совпадения (или «не нашёл»).
**19.** `Найди в Общих задачи про авто` → поиск **внутри** «Общие задачи».
**20.** `Где задача "Позвонить Иванову"?` → находит её (создавали в R2) и показывает, в каком проекте.

## Блок 5. Создание задач (одиночное)

> После каждого создания проверь, что задача реально появилась в Plane.

**21.** `Создай задачу Купить хлеб в Общих задачах` → создаётся «Купить хлеб» в «Общие задачи». *(понадобится дальше — в блоках 7–8)*
**22.** `Добавь в Специальные задачи задачу "Тестовая"` → «Тестовая» в «Специальные задачи».
**23.** `Заведи задачу Подготовить отчёт в Общих с описанием "Квартальный отчёт за Q2"` → задача + заполненное описание (проверь в карточке задачи).
**24.** `Создай задачу в Общих с эмодзи 🚀 Запуск` → задача «🚀 Запуск».
**25.** `Создай задачу ""` (пустое имя) → «Укажи имя задачи», НЕ создаёт «Без названия».
**26.** `Сделай задачу "Тест" в проекте общие задачи` (нижний регистр) → создаёт в «Общие задачи» (регистр не важен).
**27.** `Создай задачу 'Купить хлеб' в Общих` (дубликат) → создаёт **второй** «Купить хлеб» (дубликаты разрешены, агент не должен мешать).

## Блок 6. Приоритеты и сроки (умные)

> Приоритет/срок проверяй в карточке задачи в Plane.

**28.** `Создай в Общих "Прод лежит, чиним"` → приоритет **срочный** (по словам «прод лежит»).
**29.** `Создай в Общих задачу "Отчёт" с высоким приоритетом` → приоритет **высокий**.
**30.** `Когда будет время, добавь в Общие мелочь в дизайн` → приоритет **низкий** (по «когда будет время», «мелочь»).
**31.** `Создай в Общих "Обзор кода" со сроком до пятницы` → срок = ближайшая пятница (сверь дату от сегодняшней).
**32.** `Создай в Общих "Подготовка митапа" к концу месяца` → срок = последний день текущего месяца.
**33.** `Создай в Общих "Релиз" через 2 недели` → срок = сегодня + 14 дней.
**34.** `Создай в Общих "Закуп" на 25 декабря` → срок = 25 декабря (этот год, или следующий, если уже прошло).

## Блок 7. Изменение задач (используют «Купить хлеб» из теста 21)

**35.** `Переведи "Купить хлеб" в работу` → статус задачи стал **В работе**.
**36.** `Поставь высокий приоритет на "Купить хлеб"` → приоритет **высокий**.
**37.** `Переименуй "Купить хлеб" в "Купить багет"` → имя задачи изменилось на «Купить багет».
**38.** `Обнови описание "Купить багет": срочная закупка к ужину` → описание обновилось.
**39.** `Сдвинь срок "Купить багет" на завтра` → срок = завтра.
**40.** `Заверши "Купить багет"` / `Закрой задачу "Купить багет"` → статус **Готово**.
**41.** `Верни "Купить багет" в очередь` → статус **Запланировано** (todo).

> Для всех 35–41 сверяй фактический статус/поле в Plane, не только текст агента.

## Блок 8. Удаление задач

**42.** `Удали "Тест" из Общих` → задача «Тест» (тест 26) удаляется. Проверь, что её нет в Plane.
**43.** `Удали задачи Купить багет, Подготовить отчёт из Общих` → **bulk**: обе удаляются.
**44.** `Удали всё из Общих задач` → НЕ удаляет молча. Просит подтверждение / уточнение «точно всё?». ❌ Баг: молча удаляет все задачи.

## Блок 9. Несколько действий за один запрос (bulk / эпики)

**45.** `Создай 3 задачи: Альфа, Бета, Гамма в Специальных` → три задачи в «Специальные задачи» (см. R3).
**46.** `Создай "Альфа" в Общих, а "Бета" в Специальных` → две задачи в **разных** проектах.
**47.** `Создай в Общих "Демо" и сразу поставь высокий приоритет и срок на завтра` → **одна** задача со всеми полями (имя + приоритет высокий + срок завтра).
**48.** `Переведи Альфа и Бета в работу` → обе → статус В работе (bulk).
**49.** `Закрой "Альфа" и переведи "Бета" в работу` → разные статусы за один запрос (bulk).
**50.** `Создай эпик "Авторизация" в Общих с задачами Login, Register, Reset` → эпик + 3 подзадачи. Проверь в Plane структуру.

## Блок 10. Многоходовые диалоги (контекст) 🔴 КРИТИЧНО

> Делай шаги подряд в одной сессии. Тут проверяется память агента.

**51.** Создание без проекта (база регресса):
- Ввод 1: `Создай задачу Позвонить маме` → спрашивает, в каком проекте.
- Ввод 2: `Специальные задачи` → создаёт «Позвонить маме» в «Специальные задачи».
- ❌ Баг: показал список задач / переспросил снова / создал не то.

**52.** Местоимение «ей/её» после создания:
- Ввод 1: `Создай "Купить вино" в Общих` → создаёт.
- Ввод 2: `Установи ей срок на завтра` → ставит срок именно «Купить вино» (не переспрашивает, какая задача).

**53.** «Этот проект» из контекста:
- Ввод 1: `Покажи задачи проекта Общие задачи` → список.
- Ввод 2: `Создай в этом проекте "Контекстная"` → создаёт в «Общие задачи».

**54.** «Закрой её» после показа задачи:
- Ввод 1: `Покажи задачу "Позвонить маме"` → детали.
- Ввод 2: `Закрой её` → статус «Позвонить маме» → Готово.

**55.** Уточнение фильтра по контексту:
- Ввод 1: `Покажи срочные задачи` → срочные.
- Ввод 2: `А просроченные?` → переключается на просроченные (понимает, что речь о задачах).

## Блок 13. Sticky-проект (открытая доска)

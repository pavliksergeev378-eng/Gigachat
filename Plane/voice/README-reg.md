# Микрофон через .reg (двойной клик, без админа)

Самый простой способ включить голосовой ввод для обычного пользователя: дать ему
файл `Включить-микрофон.reg`. Прав администратора и PowerShell не нужно.

## Файлы рядом

| Файл | Что делает |
|---|---|
| **`Включить-микрофон.reg`** | разрешает микрофон для адреса (раздать пользователям) |
| **`Выключить-микрофон.reg`** | откат — убирает разрешение |
| `make-mic-reg.ps1` | генератор: пересоздать `.reg` под ВАШ адрес |

Готовые `.reg` сделаны под адрес **`http://192.168.0.100:8765`**.
Если у вас другой — см. «Под свой адрес» ниже (обязательно!).

## Что делает пользователь (3 действия)

1. Двойной клик по **`Включить-микрофон.reg`**.
2. На вопрос «Вы действительно хотите добавить…?» → **Да**.
3. **Полностью** закрыть браузер (все окна) и открыть заново.

Готово — в Plane-агенте кнопка микрофона спросит доступ и заработает. В обычном
браузере пользователя, с его закладками. На общем ПК — каждый пользователь делает
это под своей учёткой один раз.

## Под свой адрес (если не `http://192.168.0.100:8765`)

Адрес (origin) — РОВНО как в адресной строке браузера, когда открыт агент: схема +
хост + порт, без пути. Агенты раздаёт Caddy на порту **8765**, поэтому обычно это
`http://<ip-сервера>:8765` (или `http://имя:8765`, или `http://имя` на 80 порту).

Пересоздать файлы под нужный адрес (один раз, айтишником):
```
powershell -ExecutionPolicy Bypass -File make-mic-reg.ps1 -Origin "http://ВАШ-АДРЕС:8765"
```
Несколько адресов (например, по IP и по имени) — через запятую:
```
... -Origin "http://192.168.0.100:8765, http://giga:8765"
```
Скрипт перезапишет `Включить-микрофон.reg` и `Выключить-микрофон.reg`.

> Можно и вручную: открыть `.reg` в блокноте и заменить адрес. Но проще генератором —
> он сразу проставит адрес во все браузеры и не ошибётся с кодировкой.

## Проверка, что сработало

1. После перезапуска браузера открыть `chrome://policy` (в Яндексе —
   `browser://policy`, в Edge — `edge://policy`). Должна быть строка
   `OverrideSecurityRestrictionsOnInsecureOrigin = http://192.168.0.100:8765`.
2. F12 → Console → ввести `isSecureContext` → должно быть `true`.
3. Нажать микрофон в агенте → браузер спросит доступ → **Разрешить**.

## Если не сработало

- **`chrome://policy` пустой / нет строки** → браузер не перезапускали полностью
  (закройте ВСЕ окна, проверьте, что процесс браузера завершился), либо `.reg` не
  применился (повторите двойной клик → «Да»).
- **Адрес не совпадает** → политика привязана к ТОЧНОМУ origin. Если в адресной
  строке `http://192.168.0.100:8765`, а в политике `http://192.168.0.100` (без порта)
  — не сработает. Пересоздайте под точный адрес.
- **Откатить** → двойной клик по `Выключить-микрофон.reg` → «Да» → перезапуск браузера.

## Чем отличается от скрипта `enable-mic-http.ps1`

Это тот же приём (политика `OverrideSecurityRestrictionsOnInsecureOrigin`), но:
- `.reg` — двойной клик, без админа, в раздел текущего пользователя (HKCU);
- `enable-mic-http.ps1` — на весь ПК (все пользователи, HKLM), но требует прав админа.

Выбирайте `.reg` для простой само-установки пользователем, скрипт — когда нужно
разом на весь ПК для всех учёток.

# Голосовой ввод в Plane-агенте — запуск на всех ПК офиса (офлайн, http)

Распознавание речи (Vosk) работает **полностью офлайн** — модель и WASM раздаются с
вашего сервера, в интернет ничего не уходит.

Единственное препятствие: браузер даёт доступ к **микрофону** только в «защищённом
контексте» — это `https` **или** `localhost`. На `http://<ip-сервера>` (как у вас по
офису) микрофон заблокирован **самим браузером** — это правило безопасности, его
нельзя обойти ни со страницы, ни на сервере. Значит, на каждом ПК нужно один раз
сказать браузеру: «этому адресу доверяй как защищённому».

**Ваш выбор:** оставляем `http`, ставим политику браузера
`OverrideSecurityRestrictionsOnInsecureOrigin` на **весь ПК** (HKLM → все
пользователи). После этого любой обычный пользователь просто открывает агент и жмёт
микрофон — больше ничего настраивать не надо. Сеть — **рабочая группа** (без домена),
поэтому раскатываем скриптом по ПК.

Файлы рядом:
- `enable-mic-http.ps1` — ставит политику на ОДНОМ ПК (на весь ПК, нужен админ один раз).
- `deploy-all-psexec.ps1` — массовая раскатка на список ПК через PsExec.

---

## Шаг 0. Узнайте точный адрес (origin)

Откройте Plane-агент в браузере на любом офисном ПК и посмотрите в адресную строку.
Origin — это **схема + хост + порт**, БЕЗ пути. Примеры:

| В адресной строке | Origin |
|---|---|
| `http://192.168.0.100:8765/Agents/plane-agent.html` | `http://192.168.0.100:8765` |
| `http://192.168.0.100/Agents/plane-agent.html` | `http://192.168.0.100` |
| `http://giga:8765/...` | `http://giga:8765` |

Этот origin вписывается в скрипт. Если на разные ПК заходят по-разному (по IP и по
имени) — можно указать несколько через запятую.

---

## Способ A — по одному ПК (просто и надёжно, рекомендуется для рабочей группы)

1. Откройте `enable-mic-http.ps1` в блокноте и впишите ваш адрес в строку
   ```
   $DefaultOrigin = 'http://192.168.0.100:8765'
   ```
   (поставьте СВОЙ из Шага 0). Сохраните.
2. Скопируйте файл на флэшку или в сетевую папку, доступную с офисных ПК.
3. На **каждом** ПК: правый клик по `enable-mic-http.ps1` → **«Выполнить с помощью
   PowerShell»**. Выскочит запрос прав администратора (UAC) — нажмите **Да**.
   (Скрипт сам поднимет права и поставит политику на весь ПК.)
4. **Полностью закройте браузер** (все окна) и откройте заново.

Готово — на этом ПК микрофон работает у всех пользователей.

> Если PowerShell ругается на запуск скриптов, запустите из консоли:
> `powershell -ExecutionPolicy Bypass -File enable-mic-http.ps1`

---

## Способ B — сразу на все ПК (через PsExec)

Если ПК много и есть один общий **локальный админ-аккаунт** на всех.

1. Скачайте **PsTools** (Sysinternals), положите `psexec.exe` рядом со скриптами.
2. Создайте `computers.txt` рядом — имена или IP офисных ПК, по одному в строке:
   ```
   192.168.0.101
   192.168.0.102
   PC-BUH-01
   ```
3. Запустите с админ-машины:
   ```
   powershell -ExecutionPolicy Bypass -File deploy-all-psexec.ps1 -Origin "http://192.168.0.100:8765" -AdminUser "Admin"
   ```
   Введите пароль локального админа (спросит один раз; в файлах не хранится).
4. На целевых ПК после раскатки нужно закрыть/открыть браузер.

Скрипт копирует `enable-mic-http.ps1` на каждый ПК и запускает его как админ (HKLM).

---

## Быстрая проверка одного ПК за минуту (без админа)

Чтобы просто убедиться, что всё работает, до раскатки на всех — можно поставить
политику **только для текущего пользователя** (права админа не нужны):
```
powershell -ExecutionPolicy Bypass -File enable-mic-http.ps1 -Scope User
```
Закрыть/открыть браузер → проверить микрофон. Это «для себя»; для всех — Способ A/B.

Совсем без файлов: `chrome://flags/#unsafely-treat-insecure-origin-as-secure` →
вписать ваш origin → **Enabled** → Relaunch. (В Edge — `edge://flags`, в Яндексе —
`browser://flags`.) Это тоже только для одного браузера/пользователя.

---

## Как убедиться, что сработало

1. На ПК откройте `chrome://policy` (в Яндексе — `browser://policy`, в Edge —
   `edge://policy`). Должна быть строка:
   ```
   OverrideSecurityRestrictionsOnInsecureOrigin = http://192.168.0.100:8765
   ```
   Если строки нет — браузер не перезапускали, либо политика легла не туда (запускали
   без админа в Machine-режиме).
2. Откройте Plane-агент, нажмите F12 → Console, введите `isSecureContext` —
   должно быть `true`.
3. Создайте сессию в агенте и нажмите кнопку микрофона — браузер спросит доступ →
   **Разрешить**.

---

## Если не работает

- **Кнопка пишет «требует https или localhost»** → политика не применилась.
  Проверьте `chrome://policy`; origin должен совпадать с адресной строкой **точно**
  (тот же порт). Перезапустите браузер полностью.
- **`chrome://policy` пустой после Способа A** → скрипт запускали без прав админа и
  он не смог подняться. Запустите правым кликом → «Запуск от имени администратора».
- **Микрофон спрашивает доступ, но молчит / ошибка** → это уже не про http; смотрите,
  что выбран нужный микрофон в настройках Windows и в браузере (замок слева от адреса
  → «Микрофон»).
- **Другой браузер** → скрипт ставит политику для Chrome/Edge/Yandex/Chromium/Brave
  сразу. Если у вас экзотический браузер — скажите, добавлю путь.

---

## На будущее: «правильный» путь — HTTPS

Политика выше — рабочее исключение для изолированной офисной сети. Если захотите
сделать «по-чистому» (secure context без исключений, и сразу для всех агентов) —
раздавать агенты по `https` с локальным сертификатом Caddy (`tls internal`) и один
раз раздать его корневой сертификат на ПК. Тогда никакие политики не нужны. Скажите —
подготовлю конфиг Caddy и скрипт установки корневого сертификата.

# Документация по планировщику Plane

## 1. Общее описание

**Plane-агент** — это интеграция GigaChat с open-source системой управления проектами [Plane](https://plane.so/). Агент позволяет управлять задачами, проектами, спринтами и эпиками через чат на естественном языке (русском).

Пользователь пишет в чат «Создай задачу Позвонить клиенту в Общих задачах» — и задача реально появляется в Plane. Все операции (создание, изменение, удаление, поиск, статистика) выполняются через один унифицированный n8n-воркфлоу.

---

## 2. Архитектура

```
Юзер (браузер)                Сервер (Docker)
────────────────              ─────────────────

plane-agent.html ──HTTP──>  n8n (вебхук /plane-agent)
    │                          │
    │ (token, settings,        │
    │  action/message)         ▼
    │                       Workflow plane-agent.json
    │                          │
    │                          ├── Валидация запроса
    │                          ├── Postgres: проверка токена (auth_sessions)
    │                          ├── Plane API: загрузка списка проектов
    │                          ├── GigaChat LLM: извлечение intent (NL-режим)
    │                          │      ИЛИ
    │                          ├── Direct resolve (прямой режим, без LLM)
    │                          ├── Switch: роутинг на 30+ обработчиков
    │                          └── Plane API: выполнение действия
    │
    <──── JSON ответ ───────────
```

**Два режима работы:**
1. **NL-режим** (Natural Language) — пользователь пишет произвольный текст, LLM извлекает `action` + `params`
2. **Direct-режим** — пользователь (или фронтенд) передаёт готовый `action` + `params`, LLM не вызывается. Используется для панели «Тест без LLM» и для slash-команд

## 4. Пайплайн воркфлоу (plane-agent.json)

### 4.1 Валидация (code node)
Проверяет тело POST-запроса:
- `token` — 64 hex-символа (GigaChat-сессия)
- `workspace_slug`, `plane_url`, `plane_token` — настройки подключения к Plane
- `message` (NL-режим) или `action`+`params` (Direct-режим)
- `session_id` — идентификатор чат-сессии
- `sticky_project` — закреплённый проект (из UI доски)
- `user_name` — имя пользователя

### 4.2 Проверка токена (Postgres SQL)
```sql
UPDATE auth_sessions SET last_used_at = NOW() WHERE token = $1 AND expires_at > NOW();
SELECT s.user_id, u.username FROM auth_sessions s
  JOIN auth_users u ON u.id = s.user_id
  WHERE s.token = $1 AND s.expires_at > NOW();
```

### 4.3 Загрузка проектов (Plane API)
```
GET {plane_url}/api/v1/workspaces/{workspace_slug}/projects/?per_page=1000
```
Возвращает массив проектов для контекста LLM (чтобы знал, какие проекты доступны).

### 4.4 Direct-обход (Direct resolve)
Если запрос пришёл с `action` (Direct-режим):
- Резолвит `project_name` → `project_id` (case-insensitive match)
- Проверяет обязательность проекта для действия
- Обрабатывает placeholder'ы (`<sticky>`) — подменяет на реальный sticky-проект
- Для `digest` — роутит через выход `stats` с флагом `_digest_mode=1`

### 4.5 LLM: извлечение intent (NL-режим)
Модель GigaChat получает системный промпт (~200 строк) с:
- Правилами идентичности
- Списком доступных проектов
- Описанием всех 26 actions
- Smart-приоритизацией
- Smart-датами
- Правилами bulk/многоходовых диалогов/эпиков
- Анти-враньём (запрет на chat вместо действия)

Ответ LLM — JSON: `{"action":"...","params":{...},"text":"..."}`

### 4.6 Parse LLM (code node)
Обрабатывает ответ LLM:
- Извлекает JSON из возможных обёрток (` ```json ``` `, массивов)
- Нормализует Smart-даты (завтра, пятница, 25 декабря → YYYY-MM-DD)
- Резолвит `project_name` → `project_id`
- Обрабатывает многоходовое создание (юзер ответил проектом на вопрос «в каком проекте?»)
- Для операций над существующей задачей без проекта — ищет задачу по имени во всех проектах

### 4.7 Switch: роутинг действий
30+ выходов Switch-узла, каждый ведёт к своему обработчику Plane API.

## 5. Действия (Actions)

### Операции с задачами

| Action | Назначение | Ключевые params |
|---|---|---|
| `list_issues` | Список задач проекта | `project_name` |
| `create_issue` | Создать задачу | `project_name`, `name`, `description?`, `priority?`, `target_date?`, `start_date?` |
| `update_issue` | Обновить задачу | `project_name`, `issue_name`, `name?`, `description?`, `priority?` |
| `delete_issue` | Удалить задачу | `project_name`, `issue_name` |
| `get_issue` | Детали задачи | `project_name`, `issue_name` |
| `search_issues` | Поиск в проекте | `project_name`, `query` |
| `change_status` | Сменить статус | `project_name`, `issue_name`, `state` (todo/in_progress/done/cancelled/backlog) |
| `assign_issue` | Назначить исполнителя | `project_name`, `issue_name`, `assignees` |
| `set_deadline` | Установить срок | `project_name`, `issue_name`, `target_date` |
| `add_label` | Добавить метки | `project_name`, `issue_name`, `labels` |
| `remove_label` | Убрать метки | `project_name`, `issue_name`, `labels` |

### Комментарии и подписки

| Action | Назначение |
|---|---|
| `add_comment` | Добавить комментарий к задаче |
| `list_comments` | Список комментариев |
| `update_comment` | Изменить комментарий |
| `delete_comment` | Удалить комментарий |
| `subscribe_issue` | Подписаться на задачу |
| `unsubscribe_issue` | Отписаться от задачи |
| `list_subscribers` | Кто подписан на задачу |

### Проекты и навигация

| Action | Назначение |
|---|---|
| `list_projects` | Список проектов |
| `list_members` | Участники проекта |
| `cross_project_search` | Поиск по всем проектам (query, filter: overdue/urgent/mine/created/all/today) |
| `me` | Профиль текущего пользователя |

### Циклы, модули, эпики

| Action | Назначение |
|---|---|
| `list_cycles` | Список циклов (спринтов) проекта |
| `list_modules` | Список модулей проекта |
| `create_epic` | Создать эпик с под-задачами (children) |

### Статистика и сводки

| Action | Назначение |
|---|---|
| `stats` | Статистика (scope: summary/overdue/by_assignee/my_workload) |
| `digest` | Сводка за сегодня: создано/закрыто/истекает срок |
| `list_notifications` | Лента уведомлений |
| `list_activity` | Лента активности (scope: me/all) |

### Специальные

| Action | Назначение |
|---|---|
| `bulk` | Несколько действий в одном запросе (`actions[]`) |
| `chat` | Текстовый ответ без API-вызова (приветствия, уточнения, отказы) |

## 6. Приоритеты

| Приоритет | Значение | Smart-маркеры (автоопределение) |
|---|---|---|
| Срочный | `urgent` | «срочно», «asap», «горит», «критично», «блокер», «прод лежит», «упало», «авария», «инцидент» |
| Высокий | `high` | «важно», «приоритет», «к концу дня», «к завтра», «клиент ждёт» |
| Средний | `medium` | по умолчанию |
| Низкий | `low` | «когда будет время», «несрочно», «потом», «по возможности», «не горит», «мелочь» |

## 7. Умные даты (Smart Dates)

Бэкенд нормализует человеческие форматы в YYYY-MM-DD на этапе Parse LLM:

| Ввод | Результат |
|---|---|
| `завтра` | сегодня + 1 день |
| `послезавтра` | сегодня + 2 дня |
| `к пятнице` | ближайшая пятница |
| `через 3 дня` | сегодня + 3 |
| `через 2 недели` | сегодня + 14 |
| `к концу месяца` | последнее число текущего месяца |
| `25 декабря` | 25 декабря (текущий или следующий год) |
| `25.12.2026` | как есть |

## 9. Режимы работы

### NL-режим (основной)
Пользователь пишет сообщение → LLM парсит → действие выполняется.
```
Юзер: «Создай в Общих задачах задачу Купить хлеб с высоким приоритетом»
Агент: создаёт задачу, возвращает карточку
```

### Direct-режим (тестирование)
Фронтенд отправляет готовый `action` + `params`, LLM не участвует.
Используется:
- В панели «Тест без LLM» (6 кнопок)
- Для slash-команд
- Для внутренних вызовов из UI (доска проекта)

### Sticky-проект
Когда пользователь открывает доску конкретного проекта, его имя становится «закреплённым». Все последующие запросы без явного указания проекта используют sticky. Явное указание проекта перебивает sticky.

## 10. Многоходовые диалоги

Агент хранит контекст последних 5-10 реплик. Поддерживает:

- **Создание без проекта:** «Создай задачу X» → «В каком проекте?» → «Общие задачи» → создаёт
- **Местоимения:** «Создай Купить вино» ... «Установи ей срок на завтра» → ставит срок задаче «Купить вино»
- **Контекстные ссылки:** «Покажи задачи Общих» ... «Создай в этом проекте X» → создаёт в Общих
- **Уточнение фильтра:** «Покажи срочные» ... «А просроченные?» → переключает фильтр

## 12. Фронтенд (plane-agent.html)

### Возможности
- Чат-интерфейс с историей сообщений
- Модальное окно настроек: Plane URL, workspace slug, API token
- Панель «Тест без LLM» — 6 кнопок прямого вызова API
- Доска проекта (kanban-отображение задач)
- Голосовой ввод (Vosk оффлайн-распознавание речи)
- Slash-команды
- Тёмная/светлая тема
- Управление сессиями (несколько чатов)

### Аутентификация
Фронтенд отправляет в каждом запросе:
- `token` — GigaChat-токен (из `_shared.js` / localStorage)
- `plane_url`, `workspace_slug`, `plane_token` — настройки Plane

## 13. Voice Input (Vosk)

Голосовой ввод через оффлайн-распознавание Vosk (WebAssembly). Требует разрешения микрофона.

**Проблема:** при работе по HTTP (не HTTPS) браузер блокирует доступ к микрофону.

**Решение:** флаг Chrome `OverrideSecurityRestrictionsOnInsecureOrigin` для домена офисного сервера. Скрипты и инструкции в `Plane/voice/`:
- `enable-mic-http.ps1` — установка через PowerShell (системно)
- `make-mic-reg.ps1` — генерация .reg-файла
- `Включить-микрофон.reg` / `Выключить-микрофон.reg` — готовые файлы

## 14. Развёртывание в офисе

1. `git pull` свежий код
2. `.\import-workflows.ps1 -Filter plane-agent` — импорт воркфлоу в n8n
3. `.\activate-workflows.ps1 -Force` — активация
4. В офисном Plane: создать workspace + API token
5. Найти URL, по которому n8n видит Plane (перебор из 6 вариантов через UI)
6. Открыть GigaChat Plane → настройки → ввести URL/slug/token
7. Панель «Тест без LLM» → проверить все 6 кнопок (должно быть зелёное)

## 16. Возможные проблемы

| Симптом | Вероятная причина |
|---|---|
| «Plane не настроен» | Не заполнены URL/slug/token в настройках |
| «unauthorized» | Истёк GigaChat-токен (перезайти) |
| «Plane API недоступен» | n8n не видит Plane-контейнер (сеть Docker) |
| LLM вернул английские слова | Ошибка промпта / модель |
| «Создал», но задачи нет в Plane | LLM соврал (chat вместо create_issue) — баг промпта |
| Видно `{"action":...}` в ответе | Не отработал Parse LLM |
| «Не нашёл проект» | Имя проекта не совпадает с реальным (регистр не важен, но опечатка критична) |
| Микрофон не работает | Страница открыта по HTTP (нужен флаг Chrome) |

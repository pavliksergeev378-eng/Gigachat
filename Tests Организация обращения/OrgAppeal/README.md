# Тестовые документы для алгоритма «Организация обращения» v2

20 готовых файлов (10 `.docx` + 10 `.pdf` с идентичным содержимым) — по одному на каждую финальную ветку алгоритма (A–J).
Полный список сценариев и ожидаемых результатов — в [`База данных/OrgAppeal-Setup.md`](../../База%20данных/OrgAppeal-Setup.md), раздел 7.

## Зачем два формата

- **`.docx`** проверяет браузерный путь — JSZip распаковывает ZIP прямо на клиенте, OCR не задействуется.
- **`.pdf`** проверяет серверный путь — файл уходит на OCR-сервис `localhost:8055/extract`, далее всё как обычно.

Один сценарий должен дать **одинаковый итог** в обоих форматах — это валидация обеих веток workflow.

## Формат полей в документе

Каждая служебная записка содержит до 4 идентифицирующих полей:

```
СЛУЖЕБНАЯ ЗАПИСКА

От: <ФИО>                                       ← может отсутствовать (A)
Дата рождения: <ДД.ММ.ГГГГ>                     ← может отсутствовать (B)
Личный номер: <ХХХ-ХХХ>                         ← может отсутствовать (C)
СНИЛС: <ХХХ-ХХХ-ХХХ ХХ>                         ← может отсутствовать (C)
```

В сценариях D–J все 4 поля присутствуют корректно, различия — в БД.

## Как использовать

1. Открой **Дашборд GigaChat → Алгоритмы → Организация обращения**
2. Перетащи файл из `Tests/OrgAppeal/` в зону загрузки
3. Нажми «Обработать обращение»
4. Сверь итог с колонкой «Ожидаемый итог»

Полный прогон — **20 файлов** (10 сценариев × 2 формата).

## Перегенерация

### DOCX (без зависимостей)
```
python generate.py
```

### PDF (нужен reportlab)
```
pip install reportlab
python generate-pdf.py
```

`generate-pdf.py` импортирует `TEST_CASES` из `generate.py` — содержимое всегда **идентично** docx. Шрифт — Arial из `C:\Windows\Fonts\arial.ttf`.

## Если итог отличается между DOCX и PDF

Значит проблема в одной из ветвей извлечения:
- **DOCX прошёл, PDF нет** → проблема в OCR-сервере (упал, не читает шрифт, не отвечает)
- **PDF прошёл, DOCX нет** → проблема в JSZip-пути (не подключился `jszip.min.js`, или `extractBrowserText` сломан)

## Структура файлов

- **`.docx`** — минимально валидный OOXML (3 файла внутри ZIP: `[Content_Types].xml`, `_rels/.rels`, `word/document.xml`). Текст в UTF-8 в `<w:t>`.
- **`.pdf`** — PDF 1.4 с встроенным subset шрифта Arial для кириллицы. Текст реально вписан (не картинка). Открывается MS Word, LibreOffice, любой OCR-утилитой.

# OCR-сервер GigaChat — установка и настройка

> Локальный HTTP-сервис извлечения текста из документов.
> Полностью офлайн, работает в корпоративной LAN, без интернета.
>
> Используется в workflow'ах `organization-appeal`, `document-loader` и любых других,
> где нужно вытащить текст из PDF/DOCX/картинок. URL зашит как `http://localhost:8055/extract`.

## 1. Что это и зачем

Workflow «Организация обращения» (и другие RAG-агенты) умеют принимать `.pdf`, `.docx`, картинки и т.п. Для DOCX браузер сам извлекает текст через JSZip — **OCR-сервис для них не нужен**. Но PDF и картинки требуют отдельный сервис, который умеет:

- читать «настоящие» PDF (где текст уже есть как текстовый слой)
- распознавать **сканированные** PDF (картинка отсканированного листа, без текстового слоя)
- распознавать картинки (JPG/PNG/TIFF)

Этот сервис делает всё это полностью на офисном ПК, без отправки документов наружу.

---

## 2. Требования

**На офисном ПК (для установки и работы):**
- **Python 3.12.x** в PATH (готовый Release собран строго под 3.12; для 3.10/3.11 нужно пересобирать бандл — см. раздел 11)
- ~2 ГБ свободного места на диске
- 4+ ГБ ОЗУ (EasyOCR грузит модели в память)
- CPU x86_64 (на ARM-Windows бандл не пойдёт, нужно собирать прямо там)
- GPU **не нужен** (бандл собирается с CPU-вариантом PyTorch)

**На ПК с интернетом (только если собираешь свой бандл — см. раздел 11):**
- Python 3.10–3.12 в PATH (та же мажорная версия, что на офисном ПК)
- Интернет
- Свободное место: ~600 МБ для wheels + моделей

## 3. Архитектура

```
┌──────────────────────────────────────────────────────────────┐
│  Клиент (workflow n8n или curl)                              │
│   POST /extract   multipart: file=<binary>                   │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│  ocr-server (FastAPI + uvicorn на порту 8055)                │
│                                                              │
│  По расширению файла:                                        │
│   .pdf   → PyMuPDF (быстро)                                  │
│             ├─ есть текстовый слой? → отдаём текст           │
│             └─ нет (скан)? → рендерим страницы в PNG         │
│                                EasyOCR(ru+en) → текст        │
│   .docx  → python-docx (быстро)                              │
│   .png/.jpg/.tiff/.bmp/.webp → EasyOCR                       │
│   .txt/.md/.csv → нативное чтение (UTF-8/cp1251)             │
└──────────────────────────────────────────────────────────────┘
                              ↓
                  { "text": "...", "source": "..." }
```

**Почему именно EasyOCR, а не Tesseract:**
- EasyOCR — это pip-пакет, ставится одной командой. Не требует внешнего бинаря.
- Использует тот же PyTorch, что уже стоит в `embedding-server` — без дополнительных зависимостей.
- Точность сравнима с Tesseract на русском, иногда лучше.
- Минус — модели ~95 МБ, но они нужны один раз и едут в бандле.

## 4. Этап 1. На ПК с интернетом — скачать 3 файла с GitHub

В офисе интернета нет, поэтому весь необходимый набор тащим с GitHub на ПК с интернетом, дальше — флешкой в офис.

### Шаг 4.1. Скачать сам проект GigaChat

1. Открой: **https://github.com/Jorden-maker/GigaChat**
2. Зелёная кнопка **`Code`** → **`Download ZIP`**
3. В `Загрузки` упадёт `GigaChat-main.zip` (несколько MB) — это код проекта со всеми workflow'ами и инструкциями.

### Шаг 4.2. Скачать `ocr-wheels.zip` (Python-пакеты)

1. Открой: **https://github.com/Jorden-maker/GigaChat/releases/latest**
2. В разделе **`Assets`** скачай `ocr-wheels.zip` (~260 МБ)

Это все Python-пакеты (FastAPI, PyMuPDF, EasyOCR, PyTorch CPU и т.д.), уже собранные под Python 3.12 / Windows x64.

### Шаг 4.3. Скачать `ocr-easyocr-models.zip` (модели OCR)

В том же разделе **`Assets`** скачай `ocr-easyocr-models.zip` (~95 МБ).

Это модели EasyOCR: `craft_mlt_25k.pth` (детектор) + `cyrillic_g2.pth` (распознаватель — покрывает кириллицу и латиницу). Без них OCR картинок и скан-PDF работать не будет.

### Итог Этапа 1

В `Загрузки` лежат **три файла**:

| Файл | Размер | Что внутри |
|---|---|---|
| `GigaChat-main.zip` | ~10 MB | Код проекта (`ocr-server/`, workflow'ы, документация) |
| `ocr-wheels.zip` | ~260 MB | Python-пакеты под 3.12+win-x64 |
| `ocr-easyocr-models.zip` | ~95 MB | Модели EasyOCR (ru+en) |

**Итого ~360 МБ.** Влезает на любую флешку.

> Если на офисном ПК Python другой версии (не 3.12), `ocr-wheels.zip` из Releases не подойдёт. Тогда сначала собери свой бандл — см. раздел 11 «Опционально — собрать свой бандл».

## 6. Этап 3. На офисном ПК — установить и запустить

> **Ключевая идея:** OCR-сервер ставим в **отдельную папку** `C:\ocr-server\` (или куда угодно), **вне проекта GigaChat**. Так venv (~2 GB) и wheels не сносятся при каждом обновлении проекта.

### Шаг 6.1. Создать рабочую папку

Создай папку для сервера, например `C:\ocr-server\`.

### Шаг 6.2. Распаковать `GigaChat-main.zip` и скопировать содержимое `ocr-server/`

1. Распакуй `GigaChat-main.zip` где удобно (например `C:\temp\`).
2. Внутри получится `GigaChat-main/ocr-server/`. Скопируй **содержимое** этой папки (6 файлов: `README.md`, `requirements.txt`, `server.py`, `start.bat`, `install-offline.ps1`, `make-offline-bundle.ps1`) в `C:\ocr-server\`.

### Шаг 6.3. Положить `ocr-wheels.zip` и `ocr-easyocr-models.zip` в `C:\ocr-server\`

**Не распаковывай их вручную** — `install-offline.ps1` сам обработает оба zip-а.

Содержимое `C:\ocr-server\` после шагов 6.2 и 6.3:

```
C:\ocr-server\
├── README.md
├── requirements.txt
├── server.py
├── start.bat
├── install-offline.ps1
├── make-offline-bundle.ps1
├── ocr-wheels.zip            ← из GitHub Releases
└── ocr-easyocr-models.zip    ← из GitHub Releases
```

### Шаг 6.4. Проверить Python

```cmd
python --version
```

Должно показать `Python 3.12.x` (та же мажорная версия, что на ПК сборки Release). Если Python отсутствует или другой версии:
- Поставь Python 3.12 с **офлайн-инсталлятора** Python.org (привезти с ПК с интернетом на флешке)
- Либо собери свой бандл под нужную версию (см. раздел 11)

### Шаг 6.5. Разрешить запуск PowerShell-скриптов (один раз на ПК)

На свежем Windows запуск `.ps1` блокируется политикой по умолчанию. Открой PowerShell и выполни:

```powershell
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
```

Подтверди `Y` (Да) → Enter. Это нужно сделать **один раз** на этом ПК.

> Если этот шаг пропустить — `install-offline.ps1` упадёт с ошибкой `running scripts is disabled on this system`.

### Шаг 6.6. Запустить установщик

В `C:\ocr-server\` открой PowerShell и выполни:

```powershell
.\install-offline.ps1
```

Скрипт сам:
1. Распакует `ocr-wheels.zip` → `wheels/`
2. Распакует `ocr-easyocr-models.zip` → `easyocr_models/`
3. Создаст `venv/`
4. Поставит все Python-пакеты из `wheels/` БЕЗ интернета
5. Скопирует модели EasyOCR в `C:\models\easyocr\`

Время: 2–5 минут.

Если получишь ошибку «python не найден» — добавь Python в PATH:
- Win+R → `sysdm.cpl` → Дополнительно → Переменные среды → Path → добавить `C:\Python312\` и `C:\Python312\Scripts\`
- Закрыть и заново открыть PowerShell.

Если всё прошло — увидишь зелёный «УСТАНОВКА УСПЕШНА».

### Шаг 6.7. Запустить сервер

Двойной клик по **`start.bat`** в `C:\ocr-server\`. Откроется чёрное окно с логами:

```
=== Zapusk OCR-servera ===
Port:        8055
Langs:       ru,en
EasyOCR dir: C:\models\easyocr
Endpoint:    http://localhost:8055/extract
Status:      http://localhost:8055/status
Stop:        Ctrl+C ili zakroj eto okno
===========================

INFO:     Started server process [12345]
INFO:     Application startup complete.
INFO:     Uvicorn running on http://0.0.0.0:8055
```

Это значит — сервер работает. Окно держим открытым.

> EasyOCR грузится **лениво** — модели подтянутся только при первом обращении к OCR (а не при старте). Первый запрос на скан-PDF будет ~10–30 секунд, дальше — быстро.

### Шаг 6.8. Проверка health-check

В новом окне PowerShell:

```powershell
curl http://localhost:8055/status
```

Должно вернуть:

```json
{"status":"ok","easyocr_ready":false,"easyocr_error":null,"device":"cpu","langs":["ru","en"],"port":8055}
```

`easyocr_ready: false` — это нормально на старте, модели ещё не загружены (они грузятся при первом запросе).

### Шаг 6.9. Тест на PDF

Положи рядом со скриптом любой `.pdf` (например, из `Tests/OrgAppeal/01-ivanov-full-success.pdf`):

```powershell
curl -X POST -F "file=@01-ivanov-full-success.pdf" http://localhost:8055/extract
```

Должно вернуть JSON с полем `text`:

```json
{
  "text": "СЛУЖЕБНАЯ ЗАПИСКА\n\nОт: Иванов Иван Иванович\nТабельный номер: 10001\n...",
  "source": "pymupdf",
  "filename": "01-ivanov-full-success.pdf",
  "chars": 256
}
```

`source: "pymupdf"` — значит PDF имеет текстовый слой, OCR не понадобился. Если бы это был скан — `source` был бы `"pymupdf+easyocr"`.

### Шаг 6.10. Тест на картинку (опционально, чтобы прогреть EasyOCR)

Возьми любую фотку с текстом (можно даже скриншот документа):

```powershell
curl -X POST -F "file=@screenshot.png" http://localhost:8055/extract
```

Первый запрос будет идти 15-30 секунд (EasyOCR грузит модели в память). Дальше — секунды.

## 7. Подключение к n8n

OCR-сервер ничего специального **не требует** в n8n — его URL уже зашит в workflow'ах.

Если меняешь URL (например, перенёс на другой ПК или порт):
- В n8n → `Workflow/organization-appeal.json` → нода `OCR (text extraction)` → поле URL
- В n8n → `Workflow/document-loader.json` → аналогично
- Замени `http://localhost:8055/extract` на твой адрес

## 8. Автозапуск при старте Windows

Чтобы сервер поднимался сам после перезагрузки ПК:

### Способ 1: Папка автозагрузки (простой)

1. Win+R → `shell:startup` → откроется папка автозагрузки.
2. Создай в ней ярлык на `start.bat`:
   - Правый клик в папке → Создать → Ярлык
   - Указать путь: `C:\путь\к\ocr-server\start.bat`
   - Назвать: «OCR Server»

После перезагрузки сервер запустится автоматически.

### Способ 2: Task Scheduler (надёжнее)

1. Win+R → `taskschd.msc`
2. Создать простую задачу → «OCR Server»
3. Триггер: «При входе в систему»
4. Действие: запустить программу `C:\путь\к\ocr-server\start.bat`
5. Условия: снять «Запускать задачу только при питании от сети»

### Способ 3: NSSM (как Windows-служба, для production)

Если хочешь, чтобы сервер работал даже без входа пользователя:

1. Скачай NSSM (https://nssm.cc/) — это standalone exe.
2. Запусти `nssm install OCRServer`
3. В диалоге:
   - Path: `C:\путь\к\ocr-server\venv\Scripts\python.exe`
   - Startup directory: `C:\путь\к\ocr-server\`
   - Arguments: `server.py`
4. В Environment задать `OCR_EASYOCR_DIR=C:\models\easyocr`, `OCR_PORT=8055`.

Запустить:
```
net start OCRServer
```

## 9. Если что-то не работает

### `running scripts is disabled on this system` при запуске install-offline.ps1

Пропустил шаг 6.5. Выполни:

```powershell
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
```

Подтверди `Y`, потом снова запусти `.\install-offline.ps1`.

### `python не найден` при запуске install-offline.ps1

Python не в PATH. См. шаг 6.4.

### `OSHIBKA: papka venv ne najdena` при запуске start.bat

`install-offline.ps1` не был выполнен или упал с ошибкой. Запусти его заново и убери `pause` в конце, чтобы увидеть полный лог.

### `503 EasyOCR недоступен`

Сервер не нашёл моделей EasyOCR. Проверь:

1. Папка `C:\models\easyocr\` существует?
2. В ней есть файлы `craft_mlt_25k.pth` и `cyrillic_g2.pth`?
3. В `start.bat` переменная `OCR_EASYOCR_DIR` указывает на правильный путь?

Если моделей нет — запусти `install-offline.ps1` заново (он должен был их скопировать из `easyocr_models/`).

### `Address already in use` на порту 8055

Кто-то уже занял 8055. Проверь:
```cmd
netstat -ano | findstr :8055
```

Либо смени порт в `start.bat`:
```
set OCR_PORT=8081
```
И обнови URL в n8n workflow.

### `OSError: [WinError 126] Не найден указанный модуль` (про torch_cpu.dll)

Не установлен Visual C++ Redistributable. Скачать с домашнего ПК на флешку с https://aka.ms/vs/17/release/vc_redist.x64.exe — это инсталлятор без интернета.

### Медленно работает OCR на сканах

EasyOCR без GPU — это ~1-3 секунды на страницу. Это норма для CPU.

Если есть видеокарта NVIDIA — можно ускорить в 5-10 раз:
1. Установить CUDA Toolkit
2. В `make-offline-bundle.ps1` поменять `--extra-index-url` на `https://download.pytorch.org/whl/cu121`
3. В `start.bat` добавить `set OCR_DEVICE=cuda`
4. Пересобрать бандл

### PDF читается, но текст «обрезан»

В `start.bat` подними лимит:
```
set OCR_PDF_MAX_PAGES=200
```

Также workflow `organization-appeal` сам обрезает входной текст до 15000 символов в ноде «Извлечь текст» — для коротких документов это не проблема, для больших — может терять детали.

## 10. Тонкая настройка

Все настройки задаются через переменные окружения в `start.bat`:

| Переменная | Значение по умолчанию | Что делает |
|---|---|---|
| `OCR_HOST` | `0.0.0.0` | На каком интерфейсе слушать. `0.0.0.0` = доступен из LAN. `127.0.0.1` = только локально. |
| `OCR_PORT` | `8055` | Порт сервера. |
| `OCR_LANGS` | `ru,en` | Языки EasyOCR (через запятую). См. полный список языков EasyOCR. |
| `OCR_PDF_MIN_TEXT` | `50` | Если PyMuPDF извлёк меньше N символов — fallback на EasyOCR. Увеличь, если pretty-PDF матрично-печатные документы дают мусорные «обрывки» текста. |
| `OCR_PDF_MAX_PAGES` | `50` | Лимит страниц для OCR (защита от мега-PDF). |
| `OCR_EASYOCR_DIR` | `C:\models\easyocr` | Путь к моделям EasyOCR. Должен содержать `.pth`-файлы. |
| `OCR_DEVICE` | auto | `cpu` / `cuda`. По умолчанию — auto-detect. |

### Какие endpoint'ы поддерживаются

| Метод+URL | Что делает | Запрос | Ответ |
|---|---|---|---|
| `GET /status` | Проверка живости | — | `{"status":"ok",...}` |
| `POST /extract` | Универсальный извлекатель: PDF, DOCX, картинки, TXT — определяется по расширению | `multipart: file=<binary>` | `{"text":"...","source":"...","filename":"...","chars":N}` |
| **`POST /extract-table`** | **Детекция таблиц** на изображении или PDF. Возвращает Markdown-таблицу. Если таблица не найдена — fallback на обычный OCR. | `multipart: file=<binary>` | `{"text":"...","source":"...","filename":"...","chars":N}` |

### Что такое `/extract-table`

Новый эндпоинт, который в отличие от `/extract` **не просто читает текст построчно**, а:

1. **Находит границы таблицы** на изображении с помощью OpenCV (Canny edge detection + HoughLinesP)
2. **Определяет сетку ячеек** — кластеризует горизонтальные и вертикальные линии
3. **Распознаёт текст внутри каждой ячейки** через EasyOCR
4. **Собирает Markdown-таблицу** с разделителем заголовков

**Поддержка форматов:** PDF (сканы и с текстовым слоем), PNG, JPG, TIFF, BMP, WEBP.
**Для DOCX и TXT** используйте `/extract` — в них структура извлекается нативно через python-docx.

**Когда использовать:**
- Документы с таблицами (отчёты, ведомости, прайс-листы) — `/extract-table` сохранит структуру
- Обычные сканы документов без таблиц — `/extract` (быстрее, без оверхэда на детекцию линий)
- Если не уверены — используйте `/extract-table`: он автоматически упадёт на plain-OCR, если таблицы на изображении нет

**Пример с curl:**
```bash
curl -X POST -F "file=@price-list.png" http://localhost:8055/extract-table
```
Ответ:
```json
{
  "text": "| Наименование | Цена | Количество |\n| --- | --- | --- |\n| Бумага А4 | 450 | 100 |\n| Ручка | 80 | 500 |",
  "source": "easyocr+table",
  "filename": "price-list.png",
  "chars": 88
}
```

### Как это работает (алгоритм детекции таблиц)

1. **Препроцессинг:** Grayscale → GaussianBlur → AdaptiveThreshold → BitwiseNot
2. **Поиск линий:** Canny (50, 150) → морфология (закрытие) → HoughLinesP
3. **Кластеризация:** объединение близких линий (порог ~12px), сортировка
4. **Сетка ячеек:** пересечение каждого ряда горизонтальных линий с каждой парой вертикальных
5. **OCR ячейки:** каждая ячейка по отдельности через EasyOCR
6. **Сборка Markdown:** первая строка → заголовок, остальные → тело таблицы

Если на изображении не найдено достаточно линий (менее 2 горизонтальных и 2 вертикальных) — `/extract-table` автоматически возвращает обычный плоский текст (fallback на стандартный EasyOCR).

**Ограничения:**
- Работает с таблицами, имеющими **чёткие линии границ** (нарисованные рамки, типографские линии)
- С таблицами «без линий» (только пробельное выравнивание) детекция может не сработать — в этом случае будет fallback на plain OCR
- Для наилучшего результата используйте изображения с разрешением не менее 150 DPI

### Как обновить версию EasyOCR (когда выйдет новая)

1. На домашнем ПК — поменять версию в `requirements.txt`.
2. Удалить старые `wheels/` и `easyocr_models/`.
3. Запустить `make-offline-bundle.ps1` заново.
4. Перенести на офис, заменить файлы, запустить `install-offline.ps1`.

## Дополнительно

- **CORS:** сервер по умолчанию принимает запросы с любого origin (FastAPI не блокирует кросс-доменные multipart). Если фронт в браузере жалуется на CORS — добавь middleware в `server.py` (см. `fastapi.middleware.cors.CORSMiddleware`).
- **Логи:** пишутся в `stdout` окна `start.bat`. Если нужны файлы — перенаправь: `python server.py > ocr.log 2>&1` внутри bat-скрипта.
- **Параллельные запросы:** EasyOCR thread-safe внутри одного процесса. uvicorn в дефолте — один воркер. Если нужна параллельность — запусти с `--workers 2`, но каждый воркер будет держать модели в памяти отдельно (~500 МБ × 2).

## 11. Опционально — собрать свой бандл

Используй этот раздел **только если**:
- На офисном ПК Python другой версии (не 3.12) — готовый `ocr-wheels.zip` из Releases не подойдёт
- В `requirements.txt` появилась новая зависимость и Release ещё не пересобран
- Нужно собрать под другую архитектуру (ARM)

### На домашнем ПК с интернетом

```powershell
cd ocr-server
.\make-offline-bundle.ps1
```

Скрипт сделает:
1. Скачает все Python-пакеты (~260 МБ) в `./wheels/` — CPU-вариант PyTorch.
2. Поднимет временный venv, поставит туда EasyOCR.
3. Скачает модели EasyOCR (~95 МБ) в `./easyocr_models/`.
4. Удалит временный venv.
5. Запакует `./wheels/` в `ocr-wheels.zip`.
6. Запакует `./easyocr_models/` в `ocr-easyocr-models.zip`.

Время: 10–20 минут.

### Что делать с собранными zip-ами

Вариант A — **загрузить как новый Release** (рекомендуется, если других людей тоже надо снабжать):
```powershell
gh release create v-ocr-N "ocr-wheels.zip" "ocr-easyocr-models.zip" --title "OCR bundle vN" --notes "Python X.Y, win-x64"
```

Вариант B — **просто скопировать на флешку** и идти в офис по этапу 2 (раздел 5).

### Соответствие версии Python

Бандл, собранный на Python 3.12, ставится **только** на Python 3.12.x (одинаковый major.minor). Если в офисе 3.11 или 3.10 — собирай на той же версии.

## 12. Обновление существующего сервера

Если OCR-сервер уже установлен и нужно добавить поддержку детекции таблиц (эндпоинт `/extract-table`):

### Шаг 1. Обновить server.py и requirements.txt

Скопируй новые файлы из репозитория в папку сервера (`C:\ocr-server\`):
- `server.py` — новый код с детекцией таблиц
- `requirements.txt` — добавлен `opencv-python-headless`

### Шаг 2. Доустановить opencv-python-headless

Активируй виртуальное окружение и установи новую зависимость:

```cmd
cd C:\ocr-server
venv\Scripts\activate
pip install opencv-python-headless==4.10.0.84
```

Если доступа в интернет нет — скачай пакет на домашнем ПК:
```cmd
pip download opencv-python-headless==4.10.0.84 -d wheels
```
Перенеси папку `wheels/` на офисный ПК и установи:
```cmd
venv\Scripts\pip install --no-index --find-links=wheels opencv-python-headless
```

### Шаг 3. Перезапустить сервер

Закрой окно `start.bat` и открой заново.

### Шаг 4. Проверить

```powershell
curl -X POST -F "file=@table-doc.png" http://localhost:8055/extract-table
```

Должен вернуться JSON с полем `text`, содержащим Markdown-таблицу.
Если на изображении таблицы нет — вернётся обычный плоской текст (source: `easyocr`).

3. Детальные потоки данных (по сценариям)

3.1. Чат с LLM

Браузер → GET chat-agent.html (Caddy) → браузер загружает страницу
Браузер → POST https://gigachat.devices.sberbank.ru/api/v1/chat/completions
  Request:  {"model":"GigaChat","messages":[{"role":"user","content":"Привет"}]}
  Response: {"choices":[{"message":{"role":"assistant","content":"Здравствуйте!"}}]}

3.2. Распознавание документа (OCR)

Браузер → GET text-extractor.html (Caddy) → загрузка страницы
Браузер → POST http://130.100.94.119:8055/extract [multipart: file=@scan.pdf]
  Response: {"pages":5,"text":"Извлеченный текст...","ocr_used":true,"processing_time_ms":3200}
Браузер отображает текст пользователю

3.3. n8n Workflow (импорт из Node Editor)

Node Editor → экспорт в JSON → импорт в n8n UI

Workflow:
  Webhook (POST /gigachat-workflow)
    ↓ {"message":"...", "session_id":"...", "token":"..."}
  Code (Парсинг запроса)
    ↓ {"valid":1, "text":"...", "session_id":"...", "token":"..."}
  HTTP Request (агент, через Caddy)
    ↓ {"response":"..."}
  Respond to Webhook
    ↓ {"response":"..."}

Цепочка строится ПО СВЯЗЯМ с холста Node Editor.

3.4. Экспорт плана из Plane.so

Plane Agent → GET tasks from Plane.so API (за период)
Plane Agent → генерация .docx (JSZip на стороне браузера)
Браузер → скачивание plan-YYYY-MM-DD.docx
  Формат .docx: таблица с днями, темами, расписанием задач

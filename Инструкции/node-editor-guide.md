# 📘 Конструктор агентов (Node Editor) — полная документация

**Файл:** `Agents/node-editor.html`  
**URL:** http://130.100.94.119:8765/node-editor

---

## 🔷 Что это такое

Визуальный drag-and-drop конструктор для сборки цепочек AI-агентов.  
Каждый блок на холсте — отдельный агент (чат, RAG, комплаенс, OCR и т.д.).

Результат сборки можно:
- сохранить как файл (кнопка 💾)
- загрузить ранее сохранённую схему (кнопка 📂)
- **экспортировать в n8n** (кнопка ⚡ Import в n8n)

---

## 🔷 Как пользоваться

### 1. Добавить блок
Перетащи блок из **палитры слева** на холст.  
Каждый блок — это один агент с определённым типом.

### 2. Соединить блоки
Каждый блок имеет:
- **Выходную точку (•)** справа
- **Входную точку (•)** слева

Чтобы соединить: зажми ЛКМ на **выходной точке** одной ноды и перетащи на **входную точку** другой.  
Появится линия-стрелка — соединение создано.

**Как удалить соединение:** наведи на линию → появится красный крестик → кликни.

### 3. Настроить блок
Кликни по блоку → откроется панель настроек справа.  
Можно изменить:
- **Название** (например, «Проверка комплаенса»)
- **Параметры** (зависит от типа агента)

### 4. Удалить блок
Кликни по блоку → в правой панели кнопка **🗑 Удалить ноду**.

### 5. Сохранить / Загрузить схему
- **💾 Сохранить** — скачивает JSON-файл со всеми блоками и соединениями
- **📂 Открыть** — загружает ранее сохранённый JSON

### 6. Переключить тему
Кнопка 🌙/☀️ на панели инструментов.  
Тема запоминается в браузере (localStorage).

## 🔷 Экспорт в n8n

Это главная фича. Позволяет превратить визуальную схему в работающий n8n workflow.

### Пошагово

1. **Собери цепочку** на холсте:
   - Добавь нужных агентов
   - Соедини их последовательно: выход первой ноды → вход второй, и т.д.
   - Если агентов несколько и они не связаны — каждый станет параллельной веткой

2. **Настрой параметры** каждого агента (если нужно)

3. **Нажми ⚡ Import в n8n** — скачается файл `gigachat-workflow.n8n.json`

4. **Открой n8n:** http://130.100.92.170:5678

5. **Импортируй:**
   - Workflows → Add Workflow
   - Кнопка `···` (три точки) справа сверху
   - Import from File
   - Выбери `gigachat-workflow.n8n.json`

6. В n8n появятся:
   - **Webhook** — входной узел (POST /gigachat-workflow)
   - **Парсинг запроса** — Code-узел для нормализации входа
   - **Твои агенты** — последовательно, как на холсте
   - **Ответ** — Respond to Webhook (выход)

### Что генерируется

Пример для цепочки: `АгентА → АгентБ → АгентВ`

```
Webhook → Code(Парсинг) → АгентА → АгентБ → АгентВ → Respond
```

Если ноды не соединены на холсте:
```
Webhook → Code(Парсинг) → [АгентА, АгентБ, АгентВ] (параллельно)
                          → Respond
```

### Формат выходного JSON

Файл совместим с n8n форматом импорта.  
Содержит `nodes[]`, `connections{}`, `settings`.

## 🔷 Устройство (для разработчика)

- **Холст:** `<canvas>` с поддержкой скролла (wheel) и перетаскивания
- **Ноды:** HTML-элементы внутри canvasViewport, позиционируются через `left/top`
- **Соединения:** SVG `<path>` с изогнутыми линиями (квадратичные кривые Безье)
- **Сохранение:** JSON с `nodes{}` и `connections[]`
- **Тема:** CSS custom properties (`--bg`, `--text`, `--border`, ...)
- **Зависимости:** `_config.js` + `_shared.js` (webhook URL'ы)

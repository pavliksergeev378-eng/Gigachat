# Инструкция: развернуть OCR-сервер на рабочем ПК (без интернета)

## Что у вас есть в папке проекта

```
GigaChat-main/
├── ocr-server/              ← папка с OCR-сервером (копируем на работу)
│   ├── server.py            ← сервер с детекцией таблиц
│   ├── requirements.txt     ← список зависимостей
│   ├── start.bat            ← запуск сервера
│   ├── install-offline.ps1  ← установщик (офлайн)
│   ├── README.md            ← документация
│   ├── make-offline-bundle.ps1 ← сборщик бандла (не нужен на работе)
│   ├── wheels.zip           ← 256 МБ — Python-пакеты
│   └── easyocr_models.zip   ← 94 МБ — модели EasyOCR
└── НА-РАБОТУ-ИНСТРУКЦИЯ.md  ← этот файл
```

---

## Пошаговая инструкция

### Шаг 1. Перенести папку на рабочий ПК

Скопируйте папку **`ocr-server`** целиком (через флешку или сетевую папку) на рабочий ПК, например в `C:\ocr-server\`.

**Важно:** обе папки `wheels` и `easyocr_models` уже лежат ВНУТРИ `ocr-server`.

```
C:\ocr-server\
├── server.py
├── requirements.txt
├── start.bat
├── install-offline.ps1
├── README.md
├── wheels.zip              ← ЕСТЬ? отлично
└── easyocr_models.zip      ← ЕСТЬ? отлично
```

---

### Шаг 2. Проверить Python

Откройте **PowerShell** (пуск → PowerShell) и выполните:

```powershell
python --version
```

Должно показать `Python 3.12.x` (или 3.10–3.12).  
Если Python не установлен — возьмите установщик с домашнего ПК на флешке.

---

### Шаг 3. Разрешить запуск PowerShell-скриптов (один раз)

```powershell
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
```

Нажать `Y` → Enter.

---

### Шаг 4. Запустить установщик

В PowerShell перейдите в папку сервера и запустите:

```powershell
cd C:\ocr-server
.\install-offline.ps1
```

Установщик сам:
1. Распакует `wheels.zip`
2. Распакует `easyocr_models.zip`
3. Создаст виртуальное окружение (`venv`)
4. Установит все пакеты из wheels БЕЗ интернета
5. Скопирует модели EasyOCR в `C:\models\easyocr`

Ждите 2–5 минут. Должна появиться зелёная надпись **«УСТАНОВКА УСПЕШНА»**.

---

### Шаг 5. Запустить сервер

Двойной клик по **`start.bat`** в `C:\ocr-server\`.

Откроется чёрное окно с логами. Если видите:

```
=== Zapusk OCR-servera ===
Port:        8055
Endpoint:    http://localhost:8055/extract
Status:      http://localhost:8055/status
```

— сервер работает. Окно **не закрывайте**, оно должно быть открыто всё время.

---

### Шаг 6. Проверить, что сервер отвечает

Откройте **новое** окно PowerShell и выполните:

```powershell
curl http://localhost:8055/status
```

Должно вернуть:
```json
{"status":"ok","easyocr_ready":false,...}
```

`easyocr_ready: false` — это нормально, модели загрузятся при первом обращении.

---

### Шаг 7. Проверить обычный OCR

Возьмите любой PDF-файл (например, из папки `Tests`) и отправьте на сервер:

```powershell
curl -X POST -F "file=@C:\путь\к\файлу.pdf" http://localhost:8055/extract
```

Должен вернуться JSON с полем `text`.

---

### Шаг 8. Проверить детекцию таблиц

Возьмите изображение или PDF с таблицей:

```powershell
curl -X POST -F "file=@C:\путь\к\файлу-с-таблицей.png" http://localhost:8055/extract-table
```

Если на картинке есть таблица — вернётся Markdown-таблица:

```json
{
  "text": "| Название | Цена |\n| --- | --- |\n| Товар А | 100 |",
  "source": "easyocr+table",
  ...
}
```

Если таблицы нет — вернётся обычный текст (source: `easyocr`).

## Если что-то пошло не так

**Ошибка: `running scripts is disabled`** — не выполнен Шаг 3. Выполните `Set-ExecutionPolicy`.

**Ошибка: `python не найден`** — Python не установлен или не в PATH. Нужен Python 3.12.

**Ошибка: `не хватает wheels/`** — забыли скопировать `wheels.zip` рядом с `install-offline.ps1`.

**Ошибка: `503 EasyOCR недоступен`** — модели EasyOCR не скопировались. Проверьте, что есть папка `C:\models\easyocr\` с файлами `.pth`.

**Ошибка: `Address already in use` на порту 8055** — кто-то уже занял порт. Можно сменить порт в `start.bat` (строка `set OCR_PORT=8055` → например `set OCR_PORT=8056`).

**Сервер не видит opencv / таблицы не детектятся** — проверьте, что `opencv-python-headless` установлен:

```powershell
cd C:\ocr-server
venv\Scripts\activate
pip list | findstr opencv
```

Должен показать `opencv-python-headless 4.10.0.84`. Если нет — установите вручную из wheels:

```powershell
venv\Scripts\pip install --no-index --find-links=wheels opencv-python-headless
```

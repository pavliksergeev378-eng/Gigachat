# Тестовый набор для file-merger.html

25 файлов покрывают все фичи и edge cases инструмента «Объединение файлов».
Перегенерировать: `python _generate.py` (нужны PIL, reportlab, python-docx, openpyxl).

## Что какой файл тестирует

### Word — базовые
| Файл | Что внутри | Что проверяет |
|---|---|---|
| `word-1-только-текст.docx` | 4 параграфа | Извлечение `<w:p>` |
| `word-2-только-таблица.docx` | таблица 5×3 | Извлечение `<w:tbl>` |
| `word-3-текст-плюс-таблица.docx` | заголовок + параграф + таблица | Сохранение порядка элементов |
| `word-4-несколько-таблиц.docx` | 2 таблицы с подзаголовками | Множественные `<w:tbl>` |
| `word-5-длинный-текст.docx` | 3 главы по нескольку параграфов | Тест на отсутствие лишних разрывов (фикс «большой spacing») |

### Word — расширенные (новые фичи)
| Файл | Что внутри | Что проверяет |
|---|---|---|
| `word-6-с-картинкой.docx` | текст + встроенная картинка | **B11** — копирование media из source docx |
| `word-7-со-стилями.docx` | H1, H2, bold/italic, нумерованные/маркированные списки, цветной текст | **#5** — сохранение стилей из source |

### Excel — базовые
| Файл | Что внутри | Что проверяет |
|---|---|---|
| `excel-1-сотрудники.xlsx` | 5×3, строки + числа | Базовая конвертация |
| `excel-2-товары.xlsx` | 7×5, числа в большинстве колонок | `String(0)` правильно конвертится |
| `excel-3-финансы.xlsx` | 5×4, всё числовое | Числа в Word-таблице |
| `excel-4-контакты.xlsx` | 5×4, длинные строки | Длинные ячейки не ломают layout |
| `excel-5-расписание.xlsx` | 6×5, разные типы | Обычный multi-column |

### Excel — расширенные (новые фичи)
| Файл | Что внутри | Что проверяет |
|---|---|---|
| `excel-6-многолистовой.xlsx` | 3 листа: Сотрудники / Продажи / Контакты | **#3** — все листы в выходе, каждый со своим подзаголовком «Лист: ...» |
| `excel-7-merged-cells.xlsx` | merged заголовок над 4 столбцами + vertical-merged ячейки Q1/Q2 | **B13** — gridSpan + vMerge в выходной Word-таблице |

### PDF (новая фича #1)
| Файл | Что внутри | Что проверяет |
|---|---|---|
| `pdf-1-простой.pdf` | 1 страница, 3 параграфа | Базовое извлечение текста через pdf.js |
| `pdf-2-многостраничный.pdf` | 3 страницы | Page break между страницами в выходе |
| `pdf-3-длинный.pdf` | 1 страница с большим количеством текста | Длинный контент |

### CSV / TSV (новая фича #2)
| Файл | Разделитель | Что проверяет |
|---|---|---|
| `csv-1-простой.csv` | запятая | Автодетект `,` |
| `csv-2-с-кавычками.csv` | запятая, escaped quotes, `\n` внутри ячейки | Парсер кавычек и многострочных ячеек |
| `tsv-1-табы.tsv` | таб | Автодетект `\t` |

### Фото
| Файл | Размер | Формат | Что проверяет |
|---|---|---|---|
| `photo-1-логотип.png` | 800×600 | PNG | Базовый embed |
| `photo-2-график.jpg` | 900×500 | JPG | JPEG content-type |
| `photo-3-градиент.png` | 600×800 | PNG | Portrait ориентация |
| `photo-4-шашки.png` | 500×500 | PNG | Квадратное фото |
| `photo-5-баннер.jpg` | 1600×400 | JPG | Wide — должно автоматически уменьшиться до 14 см |

## Готовые сценарии для прогона

### Сценарий 1 — типичный кейс
Файлы: `word-3-текст-плюс-таблица.docx` + `excel-1-сотрудники.xlsx` + `photo-1-логотип.png`

**Ожидаемо:** заголовок раздела → параграф → таблица заказа → пустая строка → таблица сотрудников → пустая строка → логотип.

### Сценарий 2 — несколько листов Excel (#3)
Файлы: `excel-6-многолистовой.xlsx`

**Ожидаемо:** 3 жирных подзаголовка «Лист: Сотрудники», «Лист: Продажи», «Лист: Контакты клиентов», под каждым своя таблица.

### Сценарий 3 — merged cells (B13)
Файлы: `excel-7-merged-cells.xlsx`

**Ожидаемо:** в Word-таблице первая ячейка «Квартальный отчёт 2024» растянута на 4 столбца (`gridSpan`), ячейки Q1 и Q2 в левой колонке растянуты на 3 строки вниз (`vMerge`).

### Сценарий 4 — картинка из Word (B11)
Файлы: `word-6-с-картинкой.docx`

**Ожидаемо:** в выходном Word есть параграф «Документ со встроенной картинкой», картинка-логотип, параграф после неё. Картинка из исходного docx скопирована корректно.

### Сценарий 5 — стили из Word (#5)
Файлы: `word-7-со-стилями.docx`

**Ожидаемо:** H1 и H2 заголовки крупнее основного текста, bold/italic видны внутри параграфа, нумерованный/маркированный списки выглядят как списки, последний абзац красный 14pt.

### Сценарий 6 — PDF (#1)
Файлы: `pdf-2-многостраничный.pdf`

**Ожидаемо:** 3 раздела, между ними разрыв страницы (`<w:br type="page"/>`). Текст с кириллицей читается корректно.

### Сценарий 7 — CSV (#2)
Файлы: `csv-2-с-кавычками.csv`

**Ожидаемо:** таблица 4×3, ячейка `Цитата: "Быть или не быть"` корректно показывает внутренние кавычки. Ячейка с `\n` рендерится в две строки через `<w:br/>`.

### Сценарий 8 — большой prog test (всё сразу)
Файлы: **все 25 файлов** через «выбрать папку целиком» (#10)

**Ожидаемо:** в карточках каждого файла спиннер (#9) во время парсинга, затем галочки ✓. В превью (#8) — корректная статистика по каждому источнику. После «Собрать» — один большой docx со всем содержимым в порядке сортировки.

### Сценарий 9 — UX (#4, #6, #7, #11, #12)
1. Открыть инструмент, добавить 3 файла
2. Изменить **сортировку** на «Имя» (#12) — порядок изменится
3. В settings выбрать разделитель **«Заголовок с именем файла»** (#4)
4. Сменить **имя выходного** на `Тестовый отчёт` (#6)
5. Кликнуть «Собрать»
6. Перезагрузить страницу (Ctrl+R) — настройки должны восстановиться из **localStorage** (#7)
7. Все системные сообщения — **тосты** в правом верхнем углу (#11), не баннер в макете

### Сценарий 10 — превью без сборки (#8)
1. Добавить разнотипные файлы
2. Кликнуть **«Превью»** (не «Собрать»)
3. Открывается модалка с описанием каждого файла + summary внизу
4. Никаких файлов не скачивается, можно просто закрыть

## Перегенерация

```powershell
cd C:\Users\Lenovo\Desktop\GigaChat\Тест. Объединение файлов
python _generate.py
```

Зависимости:
- `Pillow` — для PNG/JPG
- `reportlab` — для PDF с кириллицей
- `python-docx` — для расширенных Word
- `openpyxl` — для multi-sheet и merged cells Excel

Если что-то не установлено: `pip install Pillow reportlab python-docx openpyxl`.
